获取Class实例

May 22, 2021 One-minute read

获取Class实例的四种方式 

//获取Class实例的4种方式 -- 注意是 实例
class Demo{
    public static void main(String[] args) throws ClassNotFoundException {
        //1、类名.class
        Class  clazz1=Demo.class;

        //2、对象.getClass
        //Class clazz2 =new Demo().getClass();
        Demo demo=new Demo();
        Class clazz2=demo.getClass();

        //3、Class.forName(全类名)  --带路径
        Class clazz3= Class.forName("com.flipped.test1.Demo");

        //4、类加载器
        ClassLoader classLoader =Demo.class.getClassLoader();
        Class clazz4 = classLoader.loadClass("com.flipped.test1.Demo");

        System.out.println(clazz1);
        System.out.println(clazz2);
        System.out.println(clazz3);
        System.out.println(clazz4);
    }
}

常用的三种方法：1、2、3。

评价：

// 第一种方法：任何数据类型都有一个静态属性.class类获取其对应的Class对象。相对简单 但还是要明确用到类中的静态成员。

//第二种方法：Object类中的getClass()方法，想要使用这种方式，必须要明确具体的类，并创建对象，比较麻烦

//第三种方法:只要通过给定的类的字符串名称(类路径)就可以获取该类，更为扩展。
//用Class 类中的方法就可以完成,该方法就是Class.forName(), 该方法只要有名称即可，更方便，扩张性更好。

Class.forName和ClassLoader的区别

两者最大的区别 Class.forName得到的Class实例是已经初始化完成的 Classloder.loaderClass得到的Class实例是还没有链接的

java中class.forName()和classLoader都可用来对类进行加载。 class.forName()除了将类的.class文件加载到jvm中之外，还会对类进行解释，执行类中的static块。 而classLoader只干一件事情，就是将.class文件加载到jvm中，不会执行static中的内容,只有在newInstance才会去执行static块。 Class.forName(name, initialize, loader)带参函数也可控制是否加载static块。并且只有调用了newInstance()方法采用调用构造函数，创建类的对象。

应用场景：

在我们熟悉的Spring框架中的IOC的实现就是使用的ClassLoader。

而在我们使用JDBC时通常是使用Class.forName()方法来加载数据库连接驱动。这是因为在JDBC规范中明确要求Driver(数据库驱动)类必须向DriverManager注册自己。
